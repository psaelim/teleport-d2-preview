What does not exist yet

The honest distance between a beautiful prototype and a shippable patient app, grouped by journey. Anything amber already has a visible button waiting for it.

Missing Account & onboarding

  • Sign-up form name, date of birth, contact — the new-patient path behind "Continue"
  • Verification code entry the sign-in screen promises a texted code
  • Identity verification photo ID — telehealth compliance requires it
  • Medical intake history, medications, allergies — once, not as a nag
  • Consents telehealth consent + privacy, in plain language
  • Permission priming camera, mic, notifications, location — asked in context, not at launch
  • Account recovery new phone number, lost access

Stubbed Insurance & money

  • Insurance card capture Home banner + You row both promise it
  • Eligibility check result the old app's best flow — worth carrying over
  • Coverage breakdown "$50 or covered" needs a screen that explains which
  • Payment methods add / change card; referenced on Price and You
  • Billing history receipts — the notification links to a PDF that isn't there

Stubbed Clinical surfaces

  • Rx refill flow a quarter of the Get care screen
  • Pharmacy selection where the refill goes
  • Medication list Atorvastatin exists only inside the visit summary
  • Records library documents beyond lab results
  • Results trends the "Trends" tab is a stub; needs charts over time
  • Pre-visit intake "reason for visit" + photo upload before the call

Missing Visit lifecycle

  • Cancel / reschedule tracking + booking fine print reference both
  • Call-drop recovery reconnecting state; "we'll call you back" fallback
  • In-call chat & photo share show the doctor the rash, not describe it
  • Interpreter / accessibility language and captions preferences before the call
  • Directory search "Find someone new" — never by NPI

Missing Draw logistics

  • Prep instructions fasting rules the night before, in the patient's words
  • Address book "change address" on Collect is a stub
  • Crew contact call / message buttons on Tracking are stubs
  • Specimen confirmation "dropped at the lab" — the patient's end of chain of custody

Missing System & states

  • Settings notification preferences, language, privacy
  • Help & support a human to reach when something breaks
  • Loading / error / empty / offline every screen currently assumes success
  • Provider persona everything — next per the plan
  • Crew persona everything — incl. the unsolved chain of custody
Statuses are read from the working prototype as of 15 July 2026. Old-app findings come from the 14 July audit of the existing product design file. Stub = a control that currently opens a placeholder in the prototype. Prepared by June Saelim; built with the design team's working prototypes.
